Specifications

API Usage

1. Ably Pub/Sub Channels API

Used to facilitate instantaneous communication between the client and server. This allows users to receive real-time updates on their order status, including order confirmation, preparation updates, and delivery notifications.

  • Read more

2. Google Maps Directions and Geocoding API

The Geocoding API was used to convert user-entered addresses into geographic coordinates (latitude and longitude). This functionality is essential for accurately locating users' delivery addresses and allows the system to process orders based on precise locations.

The Directions API was employed to calculate optimal routes for delivery drivers based on real-time traffic conditions. This feature enables the system to provide estimated delivery times and suggests the most efficient path for drivers to reach customers quickly.

SOLID Principles

1. Single Responsibility Principle: In the program, each class has exactly one responsibility and only one reason to change.

2. Dependency Inversion Principle: High-level modules do not depend on low-level modules; both depend on abstractions. Also, Abstractions do not depend on details, instead details depend on abstractions.

3. Open/ Closed Principle: Open for extension, closed for modifications.
